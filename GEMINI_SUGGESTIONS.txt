Stock & Spoon — Production Readiness & UX Audit
🔴 BLOCKERS (Must fix before production)
1. Console logs expose user data
100+ console.log calls remain in production code. The worst offender is in App.tsx where user IDs
 and emails are logged during household invite checks. Use logService.ts (already in the project!)
  throughout — it already gates by environment.

2. Auth state race condition
PriceDataCacheService.loadPriceData() can be called multiple times on rapid auth state changes in
 useDataManagement.ts. Needs a guard flag similar to the mealPlanSyncInProgress pattern.

3. mealPlanSyncInProgress never resets on error
If the meal plan Firestore listener fails, the global lock flag is never cleared — all subsequent
 syncs block permanently until the user refreshes. Needs a try/finally block.

4. No timeout on Gemini/image analysis
Users can wait indefinitely if the network hangs during AI image analysis. A 30-second timeout
 with a "Try again" fallback is needed.

5. No rate limiting on Gemini / Spoonacular
Any user can trigger dozens of AI requests in rapid succession. Need per-user per-minute quota
 tracking.

6. Unhandled .then() chains without .catch()
Several clipboard and Firebase redirect operations in ShoppingList.tsx and App.tsx silently 
swallow errors.

🟠 HIGH PRIORITY
Offline/Caching

Offline queue has no max size — can grow unboundedly if user is offline for extended periods
100ms debounce on sync may not prevent duplicate inventory updates across multiple tabs
Stale cache not deleted when version increments (old and new both exist)
IndexedDB quota exceeded is not handled gracefully
Error Handling

User document creation failure is silent — subsequent logic assumes the doc exists
Cache read failure and cache miss return identical results — UI can't distinguish or retry
Generic "Unable to connect" for API errors gives users no actionable hint
Performance

Promise.all() in image fetching can spawn 100+ concurrent requests on mount
All pantry/recipe items loaded into state even with react-window — 1000+ items will cause stalls
Analytics events fired on every micro-interaction (tab switch, etc.) — may hit Firebase quota
🟡 UI/UX IMPROVEMENTS
Loading & Empty States (Inconsistent)

PantryScanner briefly shows an "empty" state before items load — causes jarring flash
Shopping list and pantry are missing skeleton loaders in several paths
Meal planner shows no "no more recipes" message at the end of a list
ItemDetailModal fails silently when nutrition fetch errors — show a message instead
Modals & Focus

Modals (ItemDetailModal, RecipeModal, HouseholdInviteModal) open without trapping focus — users 
can tab into the background
No focus restoration to the trigger element when a modal closes
This also fails basic accessibility requirements
Error States

Recipe save/delete/rate failures are caught but never surfaced to the user — add a toast
Quota exceeded errors don't show an upgrade CTA — missed conversion opportunity
Forms

Validation errors not announced to screen readers (role="alert" / aria-live="polite" missing on
 error messages in Login.tsx and ShoppingList.tsx)
No visual feedback on which field failed for multi-field forms
Dark Mode

Theme toggle works but doesn't respect the OS system preference on first load — it defaults to
 light until the user manually changes it in Settings
Camera / Barcode UX

If camera permission is denied, there's no "Retry" button or link to app settings — user is stuck
Barcode detection shows a generic alert on failure — consider an inline message with next steps
